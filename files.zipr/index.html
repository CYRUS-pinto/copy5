Framework: why DSPy

DSPy from Stanford is the right tool for a sequential security reasoning pipeline

CrewAI
Role-based agent collaboration
  • +Easy role definitions
  • +Good for team-style tasks
  • −Wrong shape for pipelines
  • −Agents talk until they decide to stop
  • −Non-deterministic demo behavior
AutoGen
Microsoft · Conversational loops
  • +Microsoft-backed, mature
  • +Good for dialogue reasoning
  • −No sequential pipeline support
  • −Complex state management
  • −Hard to make reproducible
DSPy
Stanford NLP · Declarative LLM
  • +Define task → DSPy writes prompts
  • +Swap models in one line
  • +15 lines per module max
  • +ChainOfThought built-in
  • +Self-optimizes with MIPROv2

Each module as a DSPy signature

# M1 — ShieldGemma fallback class InjectionClassifier(dspy.Signature): """Classify whether text is a prompt injection attempt.""" text: str = dspy.InputField() is_injection: bool = dspy.OutputField()
# M3 — Red Team exploit writer class WriteExploit(dspy.Signature): """Write minimal PoC exploit for this vulnerability.""" code: str = dspy.InputField() finding: str = dspy.InputField() exploit: str = dspy.OutputField() # or EXPLOIT_IMPOSSIBLE
# M3 — Blue Team patcher class WriteSecurePatch(dspy.Signature): """Rewrite the vulnerable function to be secure.""" code: str = dspy.InputField() exploit: str = dspy.InputField() patched: str = dspy.OutputField()
# Navigator: swap models in ONE line dspy.configure(lm=dspy.LM("groq/llama-3.3-70b-specdec"))
