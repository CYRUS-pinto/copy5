Framework selection — why DSPy
CrewAI
Role-based agent crews
  • +Easy role definitions
  • +Good for collaborative tasks
  • −Wrong shape for pipelines
  • −Non-deterministic termination
  • −Hard to demo predictably
AutoGen
Microsoft · Conversational loops
  • +Microsoft-backed, mature
  • +Good for dialogue tasks
  • −No sequential pipeline support
  • −Complex state management
  • −Non-reproducible for demos
DSPy
Stanford NLP · Declarative LLM
  • +Define task — DSPy writes prompts
  • +Swap models in one line
  • +15 lines per module max
  • +ChainOfThought built-in
  • +Self-optimizes with MIPROv2

Each module as a DSPy Signature
